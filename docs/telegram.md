# Telegram Bot Integration

The PKL Finder Telegram interface uses the `python-telegram-bot` v21.3 library. It runs asynchronously in the main thread event loop, receiving messages from Telegram via polling.

## Command Router Configuration

All commands are defined and bound to handlers in `app/bot/handlers.py`. Registering commands with the bot instance is performed inside the `setup_bot(application)` method:

* `/start` - Displays a greeting message and explains the bot's purpose.
* `/help` - Lists all commands and usage details.
* `/search` - Runs a manual crawling cycle, matches items against the candidate profile, and sends new recommendations.
* `/latest` - Lists the 5 newest jobs flagged as recommended in the database.
* `/profile` - Displays the CV details used by the AI evaluator.
* `/stats` - Displays database statistics including the distribution of vacancies per source portal.
* `/recheck` - Triggers a manual recheck of all vacancies currently stored in the database.
* `/settings` - Displays the current active configurations loaded from `.env` (excluding full credentials).
* `/history` - Lists the 10 last actions taken by the bot.
* `/favorites` - Lists all jobs marked as favorites by the user.

## Administrative Access Constraints (Security)

Because this bot manages scrapers and database writes, access is restricted to the administrator. The `TELEGRAM_ADMIN_ID` configuration variable in `.env` specifies the only user allowed to send commands to the bot.

Security is enforced using the `@admin_only` decorator in `app/bot/handlers.py`:

```python
def admin_only(func):
    @wraps(func)
    async def wrapper(update: Update, context: ContextTypes.DEFAULT_TYPE, *args, **kwargs):
        if not update.effective_user:
            return
        
        user_id = update.effective_user.id
        if user_id != settings.TELEGRAM_ADMIN_ID:
            logger.warning(f"Unauthorized access attempt by user ID {user_id}")
            await update.effective_message.reply_text(
                "⛔️ <b>Akses Ditolak:</b> Anda tidak diizinkan menggunakan bot ini.",
                parse_mode="HTML"
            )
            return
        return await func(update, context, *args, **kwargs)
    return wrapper
```

This decorator intercepts command requests and callback queries. If the ID does not match, the action is logged and rejected.

## Message Layouts (HTML Format)

To prevent formatting errors from unescaped characters (like dots, hyphens, and brackets in markdown), all messages use HTML format.

The job alert message layout is generated by `format_job_message(job: Job)`:

```
🚀 <b>PKL / Internship Baru Ditemukan!</b>

🏢 <b>Perusahaan:</b> <code>[Company Name]</code>
💼 <b>Posisi:</b> <b>[Job Title]</b>
📍 <b>Lokasi:</b> <code>[Job Location]</code>
🌐 <b>Sumber:</b> <code>[Portal Source]</code>
💰 <b>Gaji:</b> <code>[Salary Bracket]</code>
⏱ <b>Tipe:</b> <code>[Internship/Full-time] ([Remote/On-site])</code>

<i>[Summary text explaining the decision]</i>

• [Reason 1]
• [Reason 2]
```

## Interactive Callback Queries

Alert messages and search results include inline keyboards containing buttons. This allows the user to interact with the database without typing commands:

1. **⭐ Favoritkan / ❤️ Hapus Favorit**: Toggles the favorited status of a job posting.
2. **🔗 Buka Lowongan**: A URL button pointing directly to the job application URL.

Callbacks are handled by the `CallbackQueryHandler(callback_handler)` registration. The payload string for favorites follows the pattern `[action]:[job_id]` (e.g. `fav:42` or `unfav:42`). Upon receiving a click:
* The bot verifies the user's ID against the admin ID.
* The state is updated in the database using the use cases in `JobService`.
* The keyboard layout of the message is updated to reflect the new state (replacing "⭐ Favoritkan" with "❤️ Hapus Favorit" or vice versa).
